# Security Policy

## Supported Versions
Dieses Projekt ist noch in frühem Stadium. Security Fixes werden nur für die jeweils aktuelle Entwicklungsversion bereitgestellt.

## Reporting a Vulnerability
Bitte **keine** Sicherheitslücken als öffentliche GitHub Issues melden.

Stattdessen:
1) Nutze GitHub ""Private vulnerability reporting"", falls im Repo aktiviert.
2) Alternativ: kontaktiere den Maintainer über einen privaten Kanal (wird später ergänzt).

## What to include
- Beschreibung des Problems
- Schritte zur Reproduktion
- Impact Einschätzung
- Optional: PoC (so minimal wie möglich)

Danke für responsible disclosure.
